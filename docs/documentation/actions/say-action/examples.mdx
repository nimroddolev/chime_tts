# `chime_tts.say` Examples

The following YAML examples cover additional features provided by the [`chime_tts.say` action](./):

## Basic TTS Audio
All action calls to [`chime_tts.say` action](./) must include one or more `media_player` entity targets, and a data object with the TTS [`message`](./parameters#message) and [`tts_platform`](./parameters#tts_platform) values.

### Basic example

```
action: chime_tts.say
target:
  entity_id:
    - media_player.living_room
data:
  tts_platform: google_translate
  message: Hello world!
```
<audio controls src={useBaseUrl('/audio/examples/basic.mp3?v=1')} />

### `chime_path`
Use [`chime_path`](./parameters#chime_path) to play a chime before the TTS audio (using a 'Chime TTS' chime):

```
action: chime_tts.say
target:
  entity_id:
    - media_player.living_room
data:
  tts_platform: google_translate
  message: Hello world!
  chime_path: tada
```
<audio controls src={useBaseUrl('/audio/examples/chime_path.mp3?v=1')} class="audio" />

### `end_chime_path`
Use [`end_chime_path`](./parameters#end_chime_path) to play a chime after the TTS audio (from your media folder)
```
action: chime_tts.say
target:
  entity_id:
    - media_player.living_room
data:
  tts_platform: google_translate
  message: Hello world!
  end_chime_path: /media/sounds/doorbell.mp3
```
<audio controls src={useBaseUrl('/audio/examples/end_chime_path.mp3?v=1')} class="audio" />

### `chime_path` and `end_chime_path`
Include both [`chime_path`](./parameters#chime_path) and [`end_chime_path`](./parameters#end_chime_path) to play a chime before and after TTS audio
```
action: chime_tts.say
target:
  entity_id:
    - media_player.living_room
data:
  tts_platform: google_translate
  message: Hello world!
  chime_path: drumroll
  end_chime_path: /media/sounds/doorbell.mp3
```
<audio controls src={useBaseUrl('/audio/examples/chime_path_end_chime_path.mp3?v=1')} />

## More complex examples

### `tts_speed`
Use the [`tts_speed`](./parameters#tts_speed) parameter to increase/decrease the TTS audio speed

#### Increase speed
```
action: chime_tts.say
target:
  entity_id:
    - media_player.kitchen
    - media_player.dining_room
data:
  chime_path: bells_2
  message: The washing's done!
  tts_platform: amazon_polly
  tts_speed: 150
```

### `offset`
Use the [`offset`](./parameters#offset) parameter to shift the start of chime/TTS audio backward (to overlap) or forward (to add a delay)

#### Positive Offset (delay audio)
Using a positive [`offset`](./parameters#offset) value adds a delay between chimes and TTS audio.
```
action: chime_tts.say
target:
  entity_id:
    - media_player.kitchen
    - media_player.dining_room
data:
  chime_path: drumroll
  end_chime_path: tada
  message: The suspense is killing me
  tts_platform: google_translate
  offset: 2000
```
<audio controls src={useBaseUrl('/audio/examples/offset_delay.mp3?v=1')} />

#### Negative Offset (overlay audio)
Using a negative [`offset`](./parameters#offset) value overlays chimes and TTS audio on top of each other.
```
action: chime_tts.say
target:
  entity_id:
    - media_player.kitchen
    - media_player.dining_room
data:
  chime_path: soft
  message: It's time to go to work
  tts_platform: google_translate
  offset: -2100
```
<audio controls src={useBaseUrl('/audio/examples/offset_overlay.mp3?v=1')} />

### `language` and `tld`
You can use [`language`](./parameters#language) and [`tld`](./parameters#tld) to play TTS audio with a different language and/or accent on the [Google Translate](https://www.home-assistant.io/integrations/google_translate/) TTS platform
```
action: chime_tts.say
target:
  entity_id:
    - media_player.kitchen
    - media_player.dining_room
data:
  chime_path: tada
  message: le lavage est fait!
  tts_platform: google_translate
  language: fr
  tld: fr
```
<audio controls src={useBaseUrl('/audio/examples/language.mp3?v=1')} />

### `join_players`
Use the [`join_players`](./parameters#join_players) parameter to hear the announcement across multiple speakers at the same time (only on supported speakers).
```
action: chime_tts.say
target:
  entity_id:
    - media_player.kitchen
    - media_player.dining_room
    - media_player.bedroom
data:
  chime_path: tada
  message: The washing's done!
  tts_platform: amazon_polly
  join_players: true
  unjoin_players: true
```
<audio controls src={useBaseUrl('/audio/examples/join_players.mp3?v=1')} />


### `audio_conversion`
Use the [`audio_conversion`](./parameters#audio_conversion) parameter to specify FFmpeg arguments/presets to process the generated audio.
```
action: chime_tts.say
target:
  entity_id:
    - media_player.kitchen
data:
  chime_path: bells
  message: Audio at 50% volume
  tts_platform: google_translate
  tld: com.au
  audio_conversion: Volume 50%
```

## Message Segments
The `message` parameter can be used to specify multiple [`message segments`](./parameters#advanced-usage-segment-sequences) which can be used to construct complicated notifications of muliple chimes & TTS audio segments.

### `tts` Segment Type
Review the [tts segment type](./parameters#tts-segment-type) documentation to learn more.
```
action: chime_tts.say
target:
  entity_id: media_player.kitchen_speaker
data:
  chime_path: classical
  end_chime_path: marimba
  tts_platform: google_translate
  cache: false
  language: en
  message:
    - tts: May I please have your attention?
      cache: true
    - tts: It is officially time to move the washing to the dryer
      offset: 3000
      options:
        tld: co.uk
    - tts: VÃ¡monos!
      language: es
      options:
        tld: es
      tts_speed: 115
    - type: tts
      message: Let's go!
      tts_platform: amazon_polly
```
<audio controls src={useBaseUrl('/audio/examples/segments_tts.mp3?v=1')} />

In the above example:
- The 1st segment overrides the cache parameter, setting it to true for the segment.
- The 2nd segment adds a 3 second delay before the TTS message and overrides the domain, setting it to the UK.
- The 3rd segment overrides the language, setting it to Spanish with the domain accent of Spain, and sets the speech rate to 115%.
- The 4th segment overrides the TTS platform, setting it to Amazon Polly.

### `chime` Segment Type
Review the [chime segment type](./parameters#chime-segment-type) documentation to learn more.
```
action: chime_tts.say
target:
  entity_id: media_player.kitchen_speaker
data:
  chime_path: toast
  end_chime_path: tada
  tts_platform: google_translate
  message:
    - tts: And the winner is, drumroll please
    - chime: drumroll
      offset: 500
    - tts: You!
      offset: 0
    - chime: /config/media/sounds/my_chimes/hooray.mp3
    - tts: You get to move the washing to the dryer!
    - chime: dun_dun_dun
    - tts: Sorry. You're the only one with arms.
```
<audio controls src={useBaseUrl('/audio/examples/segments_chime.mp3?v=1')} />

Review the [delay segment type](./parameters#delay-segment-type) documentation to learn more.

```
  tts_platform: google_translate

```

### `repeat`

All message segment types can include a `repeat` parameter to specify the number of times to repeat the segment.

```
action: chime_tts.say
target:
  entity_id: media_player.macbook_air
data:
  tts_platform: google_translate
  message:
    - chime: https://www.zapsplat.com/wp-content/uploads/2015/sound-effects-three/household_clock_cuckoo_strike_001.mp3
      repeat: 3
      offset: -800
    - tts: It's 3 o'clock
```
